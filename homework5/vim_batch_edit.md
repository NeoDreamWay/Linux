# Vim 高效批量编辑实战：生成 255 条 SSH 命令

## 场景描述

需要快速生成 255 条结构相似的 SSH 命令，格式为：

```bash
ssh root@10.64.198.1 > 1.log
ssh root@10.64.198.2 > 1.log
...
ssh root@10.64.198.255 > 5.log
```

其中：
- IP 最后一段从 `1` 递增到 `255`
- 日志文件按行号范围分为 5 组：`1.log` ~ `5.log`

---

## 第一步：准备模板行

在第一行输入基础模板：

```bash
ssh root@10.64.198.1 > 1.log
```

将光标放在该行，**确保处于普通模式**（按 `Esc`）。

---

## 第二步：录制宏批量生成 255 行

依次输入以下按键（普通模式下）：

| 按键 | 作用说明 |
|------|----------|
| `qq` | 开始录制宏，存入寄存器 `q` |
| `yy` | 复制整行（Yank） |
| `p` | 粘贴到下一行 |
| `$` | 跳到行尾 |
| `T>` | 反向查找 `>`，光标停在 `>` 左侧（隔离 `> 1.log` 区域） |
| `F.` | 反向查找最后一个 `.`（定位到 IP 最后一段的小数点） |
| `l` | 右移一格，光标落在 IP 最后一段数字上 |
| `<C-a>` | 数字 +1（Ctrl + a） |
| `j` | 下移一行，回到新生成的行（方便下次循环） |
| `q` | 结束录制 |

### 执行宏

录制完成后，执行 254 次宏（已有 1 行，还需生成 254 行）：

```vim
254@q
```

或先执行一次测试：

```vim
@q
```

确认无误后再批量执行：

```vim
254@@
```

> **原理**：`@@` 重复上一次使用的宏。

---

## 第三步：按范围替换日志文件名

生成 255 行后，所有行的日志文件都是 `1.log`。接下来按行号范围替换为不同的日志文件：

### 命令 1：第 51~100 行

```vim
:51,100s/> 1.log/> 2.log/g
```

**解析**：
- `:51,100` —— 指定行号范围 51 到 100
- `s/> 1.log/> 2.log/g` —— 把 `> 1.log` 替换为 `> 2.log`，`g` 表示全局替换（每行所有匹配）

### 命令 2：第 101~150 行

```vim
:101,150s/> 1.log/> 3.log/g
```

### 命令 3：第 151~200 行

```vim
:151,200s/> 1.log/> 4.log/g
```

### 命令 4：第 201~255 行

```vim
:201,255s/> 1.log/> 5.log/g
```

---

## 完整替换命令汇总

```vim
" 第 1~50 行保持默认 > 1.log（无需操作）
:51,100s/> 1.log/> 2.log/g
:101,150s/> 1.log/> 3.log/g
:151,200s/> 1.log/> 4.log/g
:201,255s/> 1.log/> 5.log/g
```

---

## 最终效果

| 行号范围 | 日志文件 | 说明 |
|----------|----------|------|
| 1 ~ 50 | `1.log` | 默认保留 |
| 51 ~ 100 | `2.log` | 替换生成 |
| 101 ~ 150 | `3.log` | 替换生成 |
| 151 ~ 200 | `4.log` | 替换生成 |
| 201 ~ 255 | `5.log` | 替换生成 |

---

## 核心技巧总结

| 技巧 | 说明 |
|------|------|
| **宏录制 (`q` + 寄存器)** | 自动化重复操作，适合批量生成结构化数据 |
| **`<C-a>` / `<C-x>`** | 快速对数字进行 +1 / -1 操作 |
| **范围替换 (`:n,ms/old/new/g`)** | 精确控制替换范围，避免误改 |
| **`T>` / `F.`** | 反向定位字符，精确控制光标位置 |

---

## 扩展：一键完成所有替换

也可以将 4 条替换命令合并为一个 Vim 脚本执行：

```vim
:51,100s/> 1.log/> 2.log/g
:101,150s/> 1.log/> 3.log/g
:151,200s/> 1.log/> 4.log/g
:201,255s/> 1.log/> 5.log/g
```

或在 Shell 中直接用 `sed` 处理：

```bash
sed -i '51,100s/> 1.log/> 2.log/g' file.txt
sed -i '101,150s/> 1.log/> 3.log/g' file.txt
sed -i '151,200s/> 1.log/> 4.log/g' file.txt
sed -i '201,255s/> 1.log/> 5.log/g' file.txt
```
